O Flock oferece dois tipos de formulários públicos que qualquer pessoa pode preencher sem uma conta no Flock. Cada formulário está vinculado a uma URL segura baseada em token, que você gera dentro do aplicativo e compartilha com sua comunidade. As submissões fluem diretamente para o módulo correspondente — sem entrada manual do seu lado. É uma das formas mais rápidas de crescer sua base de membros e o funil de integração ao mesmo tempo.

## Link público de cadastro

O link público de cadastro é feito para o autocadastro de novos membros. Quando alguém preenche o formulário, o Flock cria um cadastro completo de membro para essa pessoa, ativo e visível imediatamente no módulo Membros.

**Como funciona:**

- Sua igreja tem uma URL única de cadastro contendo um token seguro (por exemplo, `.../public/registration/<token>`).
- Você compartilha essa URL via WhatsApp, e-mail, QR code em um cartão impresso ou em qualquer outro lugar.
- O visitante abre o link em qualquer navegador — sem app para instalar ou conta para criar.
- Ele preenche os dados. O Flock valida as informações e, ao enviar, cria um novo cadastro ativo de membro.
- A submissão respeita o limite de membros do seu plano: se o limite já foi atingido, o formulário é bloqueado e o visitante é orientado a entrar em contato com a administração da igreja.

**Onde encontrar:**<br />Vá em **Configurações** ou procure a opção de gerenciamento de links na seção de Membros do painel.

**Gerenciando o link:**<br />Você pode criar múltiplos links de cadastro, definir um número máximo de usos (após o qual o link para de aceitar submissões), definir uma data de expiração e desativar um link a qualquer momento sem excluí-lo.

## Link público de integração

O link público de integração funciona da mesma forma que o de cadastro, mas as submissões vão para o **funil de Integração** em vez do módulo Membros. É ideal para acolhimento de visitantes — quem preenche o formulário se torna um integrante com status **Em andamento**, pronto para sua equipe fazer o acompanhamento.

**Como funciona:**

- Sua igreja tem uma URL única de integração contendo um token seguro (por exemplo, `.../public/integration/<token>`).
- O visitante preenche nome, data de nascimento e contatos. Também pode selecionar a congregação esperada a partir de uma lista das suas congregações ativas.
- Ao enviar, o Flock cria um novo cadastro de integrante. Os campos **Mentor** e **Observações** não fazem parte do formulário público — sua equipe os atribui depois que o cadastro aparece no sistema.
- Como integrantes não contam para o limite de membros, este formulário nunca é bloqueado por limites do plano.

**Onde encontrar:**<br />Vá em **Configurações** ou procure a área de gerenciamento de links dentro do módulo Integração.

**Gerenciando o link:**<br />Assim como no link de cadastro, você pode definir máximo de usos, data de expiração e desativar a qualquer momento.

<Tip>
  Imprima o link de integração como um **QR code** e exiba em cartões de visitante, placas de boas-vindas ou na entrada da igreja. Os visitantes escaneiam com a câmera do celular, preenchem o formulário em menos de um minuto e aparecem imediatamente na sua lista de integração — sem qualquer intervenção da equipe.
</Tip>

## Segurança por token

Cada link público usa um **token de segurança único** embutido na URL. Só quem tem o link consegue acessar e enviar o formulário. Não há endpoints de formulário publicamente listados ou descobríveis — a posse da URL específica é o único requisito.

Se um link for compartilhado por engano em um canal indevido ou você suspeitar de mau uso, **desative-o** em Configurações. O token se torna inválido imediatamente, e novas submissões são bloqueadas. Depois é só gerar um link novo para substituir.

## Visualizando submissões

As submissões de formulários públicos aparecem **imediatamente** no módulo respectivo assim que são aceitas:

- **Submissões de cadastro** → aparecem em **Membros** como membros ativos
- **Submissões de integração** → aparecem em **Integração** como integrantes com status **Em andamento**

<Note>
  Não há fila de moderação para submissões de formulários públicos — os cadastros são criados diretamente. Se preferir revisar as submissões antes que se tornem membros plenos, o funil de integração é o caminho melhor, já que integrantes exigem conversão manual para virarem membros.
</Note>
